# 看板整体架构设计

## 1. 三层架构

### 数据源层

数据源层只负责接入 Excel 文件。默认数据源是项目根目录的 `辽宁区域_加盟商贡献表_202604（测试）.xlsx`，也可以通过 Streamlit 侧边栏上传同结构 Excel 实现热替换。

约束：

- 所有业务数据 100% 来自 Excel。
- 页面数量由 Excel 有效 Sheet 自动决定。
- Sheet 名称直接作为页面名称。
- 空 Sheet 或无法识别字段的 Sheet 不参与核心业务分析。

### 数据模型层

数据模型层位于 `data_model.py`，负责把 Excel 转换成可复用的数据对象。

核心能力：

- 自动识别单行、多行、合并单元格式表头。
- 自动清理字段名中的换行、空格和重复列名。
- 自动识别通用维度：月份、加盟商、网点、区域、公斤段、状态。
- 自动识别核心指标：贡献、单量、重量、面单费、派费、补贴、扣款、收入、成本等。
- 根据指标语义选择求和或平均口径。
- 输出 KPI、排名、分布、经营结构、异常预警、数据质量校验等通用模型。

### 视图展示层

视图展示层位于 `app.py`，仅负责布局、筛选、图表、表格和导出。

视图层不硬编码 Excel 数据，不直接计算业务规则。所有页面都使用同一套组件：

- 页面标题区
- KPI 卡片区
- 维度筛选区
- 专属分析模块区
- 明细数据表格区
- CSV 导出按钮

## 2. 复用性设计

### 页面自动生成

`list_excel_sheets()` 读取 Excel 的所有有效 Sheet，并按业务优先级排序。新增同结构 Sheet 后，无需新增页面代码。

### 指标自动适配

`analysis_config.py` 只配置通用物流行业关键词，不配置具体数值。后续 Excel 数据变化时，只要字段语义保持一致，KPI、排名、分布、异常预警都会自动更新。

### 页面模块配置

重点页面通过 `PAGE_MODULES` 定义推荐模块：

- `总表-加盟商`：排名、分布、经营结构、规模分级
- `总表-网点`：网点排名、加盟商分布、状态、区域
- `派费分析报表`：派费结构、单票成本、排名、异常
- `总表-一口价`：达成、公斤段、排名、明细下钻
- `辽宁区域贡献`：区域总览、趋势、占比、结构

未显式配置的 Sheet 自动套用通用业务明细框架。

## 3. 多页面联动

侧边栏提供统一导航与全局筛选器。筛选器按当前 Sheet 是否存在对应字段自动启用：

- 月份
- 加盟商
- 区域

筛选值保存在 Streamlit Session State 中。切换页面后，如果新页面存在同语义字段，则自动沿用筛选条件；如果不存在，则该筛选项不会影响当前页面。

## 4. 性能设计

- 使用 `st.cache_data` 缓存 Sheet 列表与已加载 Sheet 数据。
- 每个页面只加载当前 Sheet，避免 15 个 Sheet 一次性全量渲染。
- 明细表默认分页显示，支持 10 万行以上数据的浏览、筛选和导出。
- 计算在数据模型层完成，视图层只渲染结果。
